# Checklist

## 文章质量检查清单

### 内容完整性检查

- [ ] 文章标题清晰、准确，能反映核心内容
- [ ] 引言部分包含问题背景、目标读者和文章结构概览
- [ ] 核心内容深入剖析技术实现细节
- [ ] 包含最佳实践和经验总结
- [ ] 总结部分回顾核心要点并提供未来展望
- [ ] 文章字数符合预期范围（4000-7000字）

### 技术准确性检查

- [ ] 技术概念解释准确、清晰
- [ ] 架构设计符合实际项目实现
- [ ] 代码示例可编译、可运行
- [ ] 代码遵循 Go 语言编码规范
- [ ] 技术方案经过实际项目验证
- [ ] 性能数据真实可靠

### 代码示例检查

- [ ] 代码示例完整，包含必要的导入和依赖
- [ ] 代码注释清晰，解释关键逻辑
- [ ] 代码示例包含错误处理
- [ ] 代码示例包含单元测试（如适用）
- [ ] 代码示例遵循项目代码风格

### 图表质量检查

- [ ] 架构图清晰展示系统结构
- [ ] 流程图清晰展示业务流程
- [ ] 时序图清晰展示服务交互
- [ ] ER 图清晰展示数据模型
- [ ] 图表配色和字体统一、美观
- [ ] 图表包含必要的图例和说明

### 可读性检查

- [ ] 文章结构清晰，层次分明
- [ ] 段落长度适中，避免过长段落
- [ ] 使用小标题分隔不同主题
- [ ] 使用列表、表格等增强可读性
- [ ] 关键术语有清晰的定义或解释
- [ ] 避免使用过于晦涩的技术术语

### 实用性检查

- [ ] 提供可落地的技术方案
- [ ] 包含实际项目经验分享
- [ ] 提供常见问题和解决方案
- [ ] 提供性能优化建议
- [ ] 提供最佳实践建议

### SEO 优化检查

- [ ] 文章标题包含关键词
- [ ] 文章摘要包含关键词
- [ ] 使用 H1、H2、H3 标签合理组织内容
- [ ] 图片包含 alt 属性
- [ ] 代码块使用正确的语法高亮

### 发布前检查

- [ ] 文章已校对，无错别字和语法错误
- [ ] 所有链接可正常访问
- [ ] 所有图片可正常显示
- [ ] 代码示例已测试通过
- [ ] 文章格式符合发布平台要求

## 系列文章一致性检查

- [ ] 所有文章使用统一的术语和命名
- [ ] 所有文章使用统一的代码风格
- [ ] 所有文章使用统一的图表风格
- [ ] 文章之间有清晰的关联和引用
- [ ] 系列文章有清晰的目录和索引

## 特定文章检查清单

### 架构设计篇检查

- [ ] 架构图清晰展示系统整体结构
- [ ] 服务拆分原则有清晰的说明
- [ ] 技术选型有充分的理由和对比
- [ ] 部署方案有详细的配置示例

### 核心实现篇检查

- [ ] 业务逻辑有完整的代码实现
- [ ] 关键算法有清晰的解释和示例
- [ ] 异常处理有详细的说明
- [ ] 性能优化有具体的数据支撑

### 架构演进篇检查

- [ ] 版本演进有清晰的时间线
- [ ] 架构变化有详细的对比说明
- [ ] 重构经验有具体的案例分析
- [ ] 技术决策有充分的理由

### 问题解决篇检查

- [ ] 问题场景有详细的描述
- [ ] 排查过程有清晰的步骤
- [ ] 解决方案有完整的代码示例
- [ ] 预防措施有具体的建议

## 发布后检查

- [ ] 文章在发布平台显示正常
- [ ] 代码示例格式正确
- [ ] 图片显示清晰
- [ ] 链接可正常跳转
- [ ] 文章分类和标签正确

## 效果评估检查

- [ ] 阅读量达到预期目标（1000+）
- [ ] 点赞和收藏达到预期目标（100+）
- [ ] 评论和讨论达到预期目标（20+）
- [ ] 收集并整理读者反馈
- [ ] 根据反馈优化文章内容
