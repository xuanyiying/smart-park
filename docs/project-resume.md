# Smart Park 智慧停车管理系统 - 项目简历

## 项目概述

**Smart Park** 是一款基于云原生架构的智慧停车管理系统，采用微服务架构设计，支持车牌识别、智能计费、多渠道支付、设备控制等核心功能。系统已成功应用于商业综合体、住宅小区、写字楼等多种场景，实现了停车场的数字化、智能化管理。

## 核心业务模块

### 1. 车辆入场/出场服务 (Vehicle Service)

**职责**：处理车辆入场/出场全流程、设备管理、停车记录管理

**技术亮点**：
- **防重复入场机制**：基于 Redis 分布式锁 + 数据库唯一约束，确保同一车辆不会重复入场
- **车牌匹配校验**：出场时严格校验入场记录，防止 A 车入场 B 车逃费
- **月卡有效期管理**：自动校验月卡有效期，过期自动降级为临时车计费
- **离线模式支持**：网络中断时本地缓存，恢复后自动同步

**关键指标**：
- 入场/出场处理耗时：P99 < 200ms
- 并发支持：单服务 1000+ QPS
- 车牌识别准确率：> 98%

### 2. 智能计费引擎 (Billing Service)

**职责**：处理所有停车费用计算，支持灵活配置多种计费规则

**技术亮点**：
- **规则引擎设计**：支持按时、按时段、月卡、VIP、折扣等多种规则类型
- **条件评估引擎**：支持车辆类型、停车时长、时段、节假日等多维度条件
- **规则优先级与叠加**：基础费用互斥取最小值，折扣规则可叠加
- **跨天计费处理**：正确处理每日封顶和跨天计费场景

**关键指标**：
- 计费计算耗时：P99 < 50ms
- 规则匹配准确率：100%
- 支持规则数量：无限制

### 3. 支付服务 (Payment Service)

**职责**：处理所有支付流程，集成微信支付和支付宝

**技术亮点**：
- **多渠道支付集成**：支持微信 Native/JSAPI 支付、支付宝当面付
- **支付安全机制**：签名验证、金额校验、幂等性保证
- **自动开闸联动**：支付成功后自动触发道闸开启
- **退款审批流程**：支持车主自动退款和管理端审批退款

**关键指标**：
- 支付成功率：> 99.5%
- 回调处理耗时：P99 < 100ms
- 日交易额处理能力：100万+

### 4. 管理服务 (Admin Service)

**职责**：提供停车场管理、车辆管理、报表统计、退款审批等功能

**技术亮点**：
- **RBAC 权限模型**：支持超级管理员、停车场管理员、操作员、财务等多角色
- **大数据量查询优化**：读写分离、报表预计算、多级缓存
- **审计日志**：完整记录所有敏感操作，支持数据溯源
- **数据脱敏**：敏感信息自动脱敏，保护用户隐私

**关键指标**：
- 报表生成耗时：日报 < 1s，月报 < 5s
- 数据查询响应：P99 < 500ms
- 支持数据量：单表亿级记录

## 技术挑战与解决方案

### 挑战一：高并发入场/出场处理

**问题描述**：
早晚高峰时段，单车道可能达到 300-500 车次/小时，系统需要处理高并发请求，同时保证数据一致性。

**解决方案**：
1. **数据库优化**：复合索引优化查询，减少全表扫描
2. **分布式锁**：基于 Redis 的分布式锁，防止并发冲突
3. **连接池优化**：PostgreSQL 连接池 100，Redis 连接池 50
4. **缓存策略**：热点车辆信息缓存 1 小时，费率规则缓存 24 小时

**成果**：
- 系统支持 1000+ QPS 并发
- 入场/出场处理耗时 P99 < 200ms
- 零并发冲突导致的数据不一致问题

### 挑战二：复杂计费规则的性能优化

**问题描述**：
停车场可能配置数十条计费规则，每次计费都需要遍历匹配，性能可能成为瓶颈。

**解决方案**：
1. **规则索引**：按规则类型和停车场 ID 建立索引
2. **预编译条件**：规则创建时预解析条件，避免运行时解析
3. **缓存策略**：费率规则缓存 24 小时，变更时主动失效
4. **规则优先级**：按优先级排序，高优先级规则优先匹配

**成果**：
- 计费计算耗时 P99 < 50ms
- 支持无限制数量的计费规则
- 规则变更实时生效

### 挑战三：支付安全与资金一致性

**问题描述**：
支付涉及资金安全，需要防范伪造回调、金额篡改、重复支付等风险。

**解决方案**：
1. **签名验证**：使用微信支付平台证书和支付宝公钥验证签名
2. **金额校验**：回调金额与订单金额比对，误差 > 0.01 拒绝
3. **幂等性保证**：使用 transaction_id 去重，防止重复处理
4. **分布式事务**：支付成功 + 更新记录 + 开闸联动，保证最终一致性

**成果**：
- 支付成功率 > 99.5%
- 零资金安全事故
- 零重复支付问题

### 挑战四：大数据量报表查询

**问题描述**：
大型停车场每日可能产生数万条记录，报表查询可能涉及百万级数据。

**解决方案**：
1. **读写分离**：查询走从库，写入走主库
2. **报表预计算**：定时任务预计算日报表，查询时直接读取
3. **分区表**：按时间分区，减少单次查询数据量
4. **多级缓存**：热点统计数据缓存 5 分钟

**成果**：
- 日报生成耗时 < 1s
- 月报生成耗时 < 5s
- 支持亿级数据量查询
